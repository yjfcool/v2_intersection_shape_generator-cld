# 路口形态生成器 架构设计文档

> 版本: 1.0
> 工程语言: C++14
> 依赖: Eigen 3.4 (线性代数), nlohmann/json (JSON解析), ClipperLib (多边形布尔运算)
> 构建系统: CMake ≥ 3.15

---

## 1. 系统目标

为自动驾驶高精地图提供**自动化路口几何形态生成能力**，从结构化路网数据自动派生：

1. 路口内连通曲线（中心线）
2. 路口内车道边线（共享/独立）
3. 精细路口面（可内凹封闭多边形）

同时满足以下硬性需求：

- 端点G1连续 + 曲线无自相交
- 同簇线组非端点不相交（拓扑优先 + 横向倒置检测）
- 障碍物优先避障，避障与同簇非交约束综合处理
- 不穿行道路边缘
- 曲线不超出粗糙路口面
- 人行横道附近调头需跨越（基于Crosswalk面）
- 输入固有形态保留（仅穿障时可重生成）
- U型调头形态满足"直线(可选)+近似圆弧+直线(可选)"

## 4. 核心模块职责

### 4.1 IntersectionShapeGenerator (顶层调度)

负责：
- 输入数据规范化（补全enterGroupId/exitGroupId）
- 拓扑校验
- SDF场构建
- 调度三个子生成器
- 异常捕获与回滚
- 性能统计

接口：
```cpp
class IntersectionShapeGenerator {
    bool generate(const IntersectionInput&, IntersectionOutput&);
    const ValidationReport& lastReport() const;
};
```

### 4.2 ConnectivityGenerator (核心生成器)

负责：
- 全局生成顺序规划 (GlobalCoordinator)
- 同簇排序求解 (ClusterOrderSolver)
- 单条曲线生成 (generateOne)
  - 兄弟曲线构造
  - 初始曲线构造（普通/U型调头分支）
  - LBFGS优化
  - 后处理（G1强制/细分/平滑）
- 多轮拓扑修复
- 违规标注

### 4.3 ClusterOrderSolver (拓扑排序)

负责：
- 进入簇/退出簇空间排序
- 同簇配对构造
- 结构性交叉检测
- 共享端点标记
- 障碍物豁免A2标记
- O(1)哈希索引查找

数据流：
```
Connectivity[] + Lane[] + LaneGroup[]
       │
       ▼
   build()
       │
       ├──► entry_group_order_[gid] = [conn_id_sorted]
       ├──► exit_group_order_[gid]  = [conn_id_sorted]
       ├──► pairs_ = [CurvePair{...}]
       ├──► entry_cluster_rank_, exit_cluster_rank_
       └──► pair_index_ = {conn_id → [pair_idx]}
```

### 4.4 SDFField (符号距离场)

负责：
- 多边形障碍物栅格化
- 带梯度的SDF查询
- 静态缓存复用
- 局部区域更新

性能优化：
- 稠密网格 + 双线性插值
- CacheKey 哈希（roi + obstacles + cs + buf）
- 内部使用 ClipperLib 做多边形并集与缓冲

### 4.5 PenaltyCost (代价函数)

联合代价：
```
F(params) = w_smooth × ∫κ²ds
          + w_obstacle × Σ max(0, cl - sdf)²
          + w_boundary × Σ boundary_intersections
          + w_fence × Σ outside_fence_dist
          + w_cluster × cluster_violation
          + w_crosswalk × crosswalk_oblique_traversal
```

权重自适应：
```
每轮外层迭代后:
  若该项违规量 > 1e-3: 权重 ×= 1.5~2.0
  上限: obstacle≤240, boundary≤160, fence≤96, cluster≤96
```

### 4.6 LBFGSSolver (优化求解器)

实现细节：
- 双循环递归计算逆Hessian近似
- Strong Wolfe线搜索（c1=1e-4, c2=0.9）
- 历史记录m=10
- 最大80次迭代
- 梯度收敛阈值5e-4

支持冷启动 (`solve`) 和热启动 (`solveWarm`)。

### 5.2 单曲线生成决策流

```
generateOne(conn, ...)
    │
    ▼ preCheck (无解预检)
    │   ├── bfsReachable?  no → makeFallbackCurve + Infeasible
    │   ├── detectSandwich? yes → tryRelaxFence
    │   └── narrow_passage?  yes → CurveStatus::WarnA2
    │
    ▼ 检测 is_uturn_geom (t0·t1 < -0.5)
    │
    ├──► is_uturn_geom = true
    │    │
    │    ▼ crosswalkClearanceAhead (基于Crosswalk面)
    │    │
    │    ▼ solveUTurnMultiConstraint
    │    │   for scale × lat_bias × lead0_extra × handle_bias:
    │    │       makeAlignedUTurnCubic
    │    │       硬过滤: 自相交/envelope/曲率
    │    │       联合代价 = sib_x×1000 + obst×200 + fence×200
    │    │                + g1×50 + maxk×20 + shape×5
    │    │   选 cost 最小的候选
    │    │
    │    ▼ 若无有效候选 → 退回 naive arch
    │
    └──► is_uturn_geom = false
         │
         ▼ buildInitialCurve
         │   ├── 直线路径穿障? → 绕行apex构造
         │   └── makeCubicG1
         │
         ▼ adaptive alpha sweep (若初始曲线与sibling交叉)
         │
         ▼ tryShapeSafeSingleCubic
         │
    ▼ LBFGS优化 (PenaltyCost)
    │
    ▼ postProcess
    │   ├── enforceG1
    │   ├── adaptiveRefine
    │   └── elasticBandSmoothAdaptive
    │
    ▼ validate (违规信息)
    │
    ▼ ConnectivityCurve
```

## 6. 关键设计决策

### 6.1 拓扑优先于几何

簇排序与结构性交叉检测在曲线生成**之前**完成，确保：
- 生成顺序正确（内→外）
- 不可避免交叉被提前豁免（StructuralCross）
- 兄弟曲线携带正确的`expected_side`进入优化器

### 6.2 单段Bezier表达U型调头

使用单段三次Bezier编码"直+半圆+直"模式，相比多段表达：
- 优化变量更少（4控制点=8参数）
- G1连续由构造保证（非约束）
- 拱弧顶点位置可控（lead0/lead1+lateral_bias）

### 6.3 网格搜索 + 局部优化组合

U型调头先做粗粒度网格搜索（~600候选）确定拓扑正确性，
再交由LBFGS做局部精细化，避免局部最优陷阱。

### 6.4 SDF静态缓存

相同输入的SDF场复用，避免重复栅格化。
CacheKey 包含 roi/obstacles/cs/buf 全部影响因子。

### 6.5 共享端点宽跳过区

同簇曲线共享退出/进入车道时，端点附近必然汇聚，
约束在端点附近18%范围跳过，仅约束中段发散区。

### 6.6 人行横道跨越仅基于Crosswalk面

`crosswalkClearanceAhead` **仅**使用输入数据的Crosswalk多边形：
- 不使用stop_line作为代理（与Crosswalk概念不同）
- 无Crosswalk数据时返回0，不施加任何跨越约束
- 避免引入任意偏移导致同arm上调头apex深度不对称

### 6.7 多轮修复

最多3轮全局修复，每轮最多修复12条曲线：
- 第1轮：纯几何拓扑修复 (tryPureGeometryTopologyRepair)
- 第2轮：U型调头专用修复 (generateOne with allow_uturn_search=true)
- 第3轮：标记Degraded

### 6.8 八重约束的协同机制

路口曲线生成同时满足8重约束，它们相互制约：

| 约束 | 权重/优先级 | 冲突对象 | 解决方案 |
|------|-----------|---------|---------|
| G1端点连续 | 由构造保证 | 同簇非交 | G1由Bezier构造保证,簇约束通过控制点偏移 |
| 同簇非端点非交 | 20 (StructuralCross豁免) | 避障 | 权重排序:避障30 > 簇20 > 围栏6;A2弱化机制 |
| 避障 | 30 (最高,可至240) | 围栏 | 优先避障,围栏可放松tryRelaxFence |
| 不穿行道路边缘 | 10 | 同簇非交 | 数值梯度仅在违反时启动 |
| 不超出粗糙面 | 6 | 避障 | U型调头网格搜索硬过滤越界候选 |
| 人行横道跨越 | min_lead0硬约束 | 同簇非交 | 用far而非forward确保同arm一致 |
| 固定形态不变 | preserved_fixed_ids | 避障 | 仅穿障时重新生成 |
| U型调头形态 | 联合代价5×arc/c偏离 | 多约束 | solveUTurnMultiConstraint网格搜索~600候选 |

#### 6.8.1 避障 vs 同簇非交协同
- 权重优先级: 避障(30) > 簇(20) > 围栏(6)
- 避障违反时自适应放大至240,簇至多96
- `checkAndMarkA2`: 若避障导致必须交叉,标记ObstacleCross弱化簇约束至0.4×(非零,保持基本保序)

#### 6.8.2 同簇非交 vs 人行横道跨越协同
- 同arm多U型调头获得不同跨越距离 → 拱弧顶点深度不一致 → 破坏有序排列
- 解决: `crosswalkClearanceAhead` 用 `far`(远端投影)而非 `forward`(最近点投影) → 同arm所有U型调头获得一致跨越距离
- U型调头求解器同时搜索 `lead0_extra` 与 `lateral_bias`,保持两条拱弧横向有序

#### 6.8.3 G1 vs 同簇非交协同
- G1由构造保证(makeCubicG1/makeAlignedUTurnCubic)
- 优化后 enforceG1 强制投影回切向射线
- 簇约束通过控制点偏移而非切向调整,与G1正交

#### 6.8.4 固定形态 vs 避障协同
- `fixedGeometryHitsObstacle` 检测固有形态是否穿障
- 仅穿障时允许重新生成,其他场景保留
- 重新生成时仍满足其他所有约束

#### 6.8.5 U型调头多约束联合求解
`solveUTurnMultiConstraint` 网格搜索 `scale × lateral_bias × lead0_extra × handle_bias`(约600候选):

```
cost = 1000 × sib_x            // 同簇交叉数(高权重)
     + 200  × obst_pen         // 障碍物穿透
     + 200  × fence_ovf        // 围栏越界
     + 50   × (1 - g1_min)     // G1偏离
     + 20   × max(maxk-1, 0)   // 最大曲率
     + 5    × |arc/c - 1.55|   // 拱弧形态偏离
     + 0.5  × (|lat_bias|+|handle_bias|+0.3·lead0_extra)  // 紧凑度
     + 0.1  × |arc - ref_len|  // 弧长偏离
```

硬过滤: 自相交/超出envelope/曲率超物理上限 → 拒绝

#### 6.8.6 生成顺序保证
按"直行→左右转→U型调头"+"由内到外"顺序生成,确保外侧曲线生成时内侧邻居已就绪,作为兄弟传入携带正确的 `expected_side`。

## 7. 性能架构

### 7.1 空间索引

- **QuadTree**: 持久化四叉树，加速同簇兄弟曲线查询
- **PairIndex**: 哈希索引，O(1)查找CurvePair
- **SDF Grid**: 稠密网格 + 包围盒限界

### 7.2 缓存

- **SDF静态缓存**: 跨调用复用
- **PenaltyCostCache**: 每外层迭代重建，存储采样点列
- **SampledSiblingCurve**: 预采样避免重复采样

### 7.3 并行化潜力

当前为单线程，可并行化点：
- 不同priority group的曲线生成（弱依赖）
- LBFGS代价函数的项计算（smooth/obstacle/boundary/fence/cluster独立）

## 8. 扩展点

### 测试分层
