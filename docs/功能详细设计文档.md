# 路口形态生成器 功能详细设计文档

> 版本: 1.1
> 适用范围: 自动驾驶高精地图路口几何形态生成
> 输入: 车道、车道组、连通关系、障碍物、边界、停止线、人行横道、粗糙路口面
> 输出: 连通曲线(中心线)、车道边线、精细路口面

---

## 1. 功能概述

本工程实现**路口几何形态自动生成**，从结构化路网数据派生：

1. **连通曲线** — 每条连通关系（进入车道 → 退出车道）对应的Bezier中心线
2. **车道边线** — 路口内部的车道左右边界线（支持相邻车道共享）
3. **精细路口面** — 由道路边缘、车道端点围成的可内凹封闭多边形

## 2. 输入数据模型

| 数据类型 | 字段 | 说明 |
|---------|------|------|
| LaneGroup | id, role(Entry/Exit), lanes, boundaries | 同方向同侧车道组 |
| Lane | id, left_edge_id, right_edge_id, width, geometry, laneOrder | 车道，geometry末点(进入)/首点(退出)为路口端点 |
| LaneEdge | id, geometry, is_shared, shared_by | 车道边线 |
| Connectivity | id, entry_lane_id, exit_lane_id, turn_type, geometry(可选) | 连通关系，geometry为可选固有形态 |
| Obstacle | id, geometry, buffered_geometry | 多边形障碍物 |
| Boundary | id, type(RoadEdge/MedianStrip/GreenBelt/Other), geometry | 道路边界 |
| StopLine | id, geometry, associated_group_id, normal_direction | 停止线 |
| Crosswalk | id, geometry(Polygon2d), crossing_direction | 人行横道（面） |
| IntersectionArea | id, geometry, is_rough | 路口面（粗糙或精细） |

## 3. 输出数据模型

### 3.1 ConnectivityCurve（连通曲线）
```cpp
struct ConnectivityCurve {
    ConnId id;
    LaneId entry_lane_id, exit_lane_id;
    ConnTurnType turn_type;
    std::shared_ptr<BezierCurve> curve;  // 生成的Bezier曲线
    LineString2d geometry;               // 采样点列
    CurveStatus status;                  // OK/WarnA2/Degraded/Infeasible
    ViolationInfo violation;             // 违规信息
    LaneEdgeId left_edge_id, right_edge_id;
};
```

### 3.2 IntersectionArea（精细路口面）
封闭多边形，外环顺时针，可包含内凹，外环点为生成几何端点+道路边缘截取点。

## 4. 路口曲线生成 —— 核心流程与多重约束协同

曲线生成是本工程的核心，需同时满足**8 重约束**且彼此相互制约。下面按约束逐项展开实现方案及协同机制。

### 4.1 约束 1：端点G1连续约束

**含义**：曲线首点必须与进入车道末点切向G1连续；曲线末点必须与退出车道首点切向G1连续；曲线自身不允许大拐角弯折；曲线自身不能自相交。

**实现方案**：

| 环节 | 实现 |
|------|------|
| 初始曲线构造 | `makeCubicG1(p0, t0, p1, t1, alpha)` 直接生成端点切向一致的三次Bezier段；U型调头用 `makeAlignedUTurnCubic` 单段编码"直+半圆+直"模式，由构造保证G1 |
| 优化后处理 | `enforceG1(c, start_tan_dir, end_tan_dir)`：将首段ctrl[1]投影到 `p0 + λ·start_dir` 射线，末段ctrl[2]投影到 `p1 - μ·end_dir` 射线，λ/μ下限0.05避免退化段 |
| 自相交检测 | `curveSelfIntersectsBusiness(c, ep=1.0)`：对80点采样做O(N²)段对相交检测，跳过相邻段与首尾对接段，端点附近容差1.0m |

**关键代码**（`src/optimizer/penalty_cost.cpp::enforceG1`）：
```cpp
// 首段: ctrl[1] 投影到首点切向射线
lam = max((c1 - p0)·sd, 0.05);
c1 = p0 + lam * sd;
// 末段: ctrl[2] 投影到尾点切向射线
mu  = max((p1 - c2)·ed, 0.05);
c2  = p1 - mu * ed;
```

### 4.2 约束 2：同簇非端点不相交约束（核心）

**含义**：
- 同进入点（组）的线簇：除端点外不能有任何相交
- 同退出点（组）的线簇：除端点外不能有任何相交
- 该约束对线簇中**所有转向类型**生效（不仅限于调头）
- 例外：逻辑上存在交叉（拓扑倒置、不同arm交叉交通）的曲线对豁免

#### 4.2.1 簇排序（`ClusterOrderSolver::build`）

**进入簇排序**（按"由内到外 = 左调头→左转→直行→右转→右调头"）：
```
主键:  DESCENDING atan2(exit_pt - mean_entry)
       角度差 > 2° 决定排序 (CCW在前 = 左侧在前)
并列:  DESCENDING 退出车道在退出组内横向位置 (稳定同向多车道)
最后:  DESCENDING 进入车道在进入组内横向位置
```

**退出簇排序**：
```
主键:  DESCENDING entry_pt · exit_arm_left_normal
并列:  DESCENDING 退出车道在退出组内横向位置
```

#### 4.2.2 配对构造（`addPairsFromSortedCluster`）

对每个簇内排序后的id列表生成全部 `(i, j), i<j` 配对：
- `expected_side = +1`：sorted[i] 应位于 sorted[j] 左侧
- `ref_perp`：配对特定横向参考轴（mean_entry→mean_exit的左法向）
- 共享退出车道或共享进入车道时 `shared_endpoint = true`：使用更宽跳过区

#### 4.2.3 结构性交叉检测（`detectTopologicalInversions`）

| 检测场景 | 判定条件 | 处理 |
|---------|---------|------|
| 同进入组对侧退出 | `exit_lat_A × exit_lat_B < 0 且 |lat|>3m` | StructuralCross |
| 同进入组横向倒置 | `(entry_lat_A - entry_lat_B) × (exit_lat_A - exit_lat_B) < 0` | StructuralCross |
| 同进入组rank-lat倒置 | `rank_diff × lat_diff > 0` | StructuralCross |
| 同退出组rank-lat倒置 | `rank_diff × lat_diff > 0` | StructuralCross |
| U型 vs 非U型(不共享车道端点) | `a_uturn != b_uturn && !share_lane_endpoint` | StructuralCross |
| **两个U型调头共享退出车道** | **不豁免** — 可按"左进=内层浅拱、右进=外层深拱"有序排列 | 受约束(shared_endpoint=true) |

#### 4.2.4 簇约束惩罚（`PenaltyCost::evalCluster`）

三重检测确保同簇非端点不相交：

**[A] 横向边距惩罚**：
```
对每个采样点 i (跳过首尾 7% 或共享端点 18%):
  diff_i = (curve_pt_i - sib_pt_nearest) · ref_perp
  若 expected_side=+1: violation = max(0, diff_i + 0.3)
  若 expected_side=-1: violation = max(0, -diff_i + 0.3)
  cost += w × violation²
```

**[B] 符号变化惩罚**（保序）：
```
检测沿弧长方向相邻采样点的 diff[] 符号翻转
每次翻转: cost += w × 50 × depth
```

**[C] 穷举段相交检测**（跨弧分数交叉）：
```
对每对 (curve_seg_i, sib_seg_j):
  若 segmentsIntersect 且交点距所有端点 > 1.5m:
    cost += w × 50 × depth
```

#### 4.2.5 共享端点处理

共享进入车道或共享退出车道的配对，端点附近必然汇聚，约束梯度近零。使用18%端点跳过区，仅约束中段64%弧段。

### 4.3 约束 3：穿障需避障约束

**含义**：曲线必须避开障碍物，不能穿行；避障优先于其他约束（除同簇非交硬约束）。

**实现方案**：

| 环节 | 实现 |
|------|------|
| SDF构建 | `SDFField::build(roi, obstacles, cs=0.2, buf=0.4)`：栅格化障碍物为符号距离场，带缓冲 |
| 障碍物代价 | `evalObstacle`: `Σ max(0, clearance - sdf)²`，权重30.0（最高） |
| 解析梯度 | `addObstacleGrad`: 利用SDF梯度 + Bezier基函数导数，O(N×S)开销 |
| 初始曲线绕行 | `buildInitialCurve`：若直线路径穿障，从SDF采样障碍物AABB，构造绕行apex |
| U型调头避障 | `solveUTurnMultiConstraint`：网格搜索时每个候选计算 `obst_pen`，权重200加入联合代价 |
| 障碍物豁免A2 | `checkAndMarkA2`：若实际相交点位于障碍物附近，标记ObstacleCross（弱化0.4×权重，非零） |

**关键代码**（`addObstacleGrad`）：
```cpp
slack = d - obstacle_clearance;
if (slack >= 0) continue;  // 无违反
coeff = 2.0 * w * slack;   // 负值
B1 = 3*(1-t)*(1-t)*t;     // ∂c/∂ctrl[1]
grad[base+0] += coeff * (gd[0] * B1);
```

### 4.4 约束 4：不能穿行道路边缘约束

**含义**：曲线不能穿越道路边界线（RoadEdge、MedianStrip、GreenBelt）。

**实现方案**：

| 环节 | 实现 |
|------|------|
| 边界代价 | `evalBoundary`: 采样段与边界线段的相交计数，权重10.0 |
| 数值梯度 | `addNumericGrad`: 仅当代价非零时启用中心差分 |
| 围栏约束（弱化版） | `evalFence`: 曲线点超出粗糙路口面时惩罚 `pointToPolygonDist × 2.0`，权重6.0 |

### 4.5 约束 5：曲线形态不能超出粗略面约束

**含义**：曲线必须保持在粗糙路口面（fence）内。

**实现方案**：
- 围栏为输入数据的 `IntersectionArea.geometry`（粗糙面）
- 每个采样点用 `polygonContains(fence, pt)` 检查
- 越界点代价 = `pointToPolygonDist(pt, fence) × 2.0`
- 权重6.0，低于避障(30)和簇约束(20)
- U型调头网格搜索时硬过滤：`enforce_fence=true` 时拒绝越界候选

### 4.6 约束 6：人行横道附近调头需跨越约束

**含义**：调头连接开始位置附近若存在人行横道（Crosswalk面），拱弧必须跨越过人行横道后才能开始。

**实现方案**（`crosswalkClearanceAhead`）：

```cpp
// 1. 仅使用输入数据的 Crosswalk 多边形作为跨越源
//    不使用 stop_line 作为代理(与人行横道概念不同)
for (const auto& cw : input.crosswalks) {
    consider_polygon(cw.geometry);
}

// 2. 对每条人行横道边段 a-b:
//    - 将 p0 投影到段上得到 closest
//    - 计算 lateral = |d · perp(T0)|, 跳过 lateral > 4m 的段
//    - far = max((a-p0)·T0, (b-p0)·T0)  // 远端纵向投影
//    - 当 far > 0 (段任意部分在 p0 前方) 且 far ≤ 8m: 候选
//    - max_far = max(max_far, far)

// 3. 返回 max_far 作为 min_lead0 传给 makeAlignedUTurnCubic
//    强制进入延长段(p0→q0)跨越人行横道后才开始近半圆
```

**重要细节**：
- 用 `far`（线段远端投影）而非 `forward`（最近点投影），保证同arm上所有U型调头获得一致跨越距离
- 无Crosswalk数据时返回0，不施加任何跨越约束

### 4.7 约束 7：曲线输入固定形态不变约束

**含义**：若连通关系的 `geometry` 字段非空（坐标数>=2），除穿障必须避障时可重新生成外，其他场景不能修改固有形态。

**实现方案**：
- `hasFixedGeometry(conn)`: 检测 `conn.geometry.points.size() >= 2`
- `makeFixedGeometryCurve(conn)`: 将固有形态转为BezierCurve
- 生成流程入口先处理所有固定形态曲线，加入 `preserved_fixed_ids` 集合
- 后续 `buildSiblings` 时，固定形态曲线作为兄弟传入但不再重新生成
- 仅当 `fixedGeometryHitsObstacle(conn, input, sdf)` 时重新生成

### 4.8 约束 8：调头形态满足直线(可选)+近似圆弧+直线(可选)

**含义**：U型调头曲线形态应满足"直线段(进入延长，可选) + 近似圆弧(拱弧) + 直线段(退出反向延长，可选)"。

**实现方案**（`makeAlignedUTurnCubic`）：

单段三次Bezier编码"直+半圆+直"模式：
```
ctrl[0] = p0
ctrl[1] = p0 + (lead0 + arc_handle) × T0 + lat_shift
ctrl[2] = p1 - (lead1 + arc_handle) × T1 + lat_shift
ctrl[3] = p1
```

参数计算：
- **lead0/lead1**（进入/退出直线延长）：
  - `s0 = p0·axis`, `s1 = p1·axis`, `common_s = max(s0, s1)`
  - `lead0 = max(0, (common_s - s0) / T0·axis)`
  - `lead1 = max(0, (common_s - s1) / (-T1)·axis)`
  - 使拱弧顶点 q0=p0+lead0·T0 与 q1=p1+lead1·(-T1) 沿U型轴线对齐

- **arc_handle**（半圆弧切向把手）：
  - `base_coef = 2/3`（经典4/3·r 半圆Bezier近似最优值，r=turn_gap/2）
  - `turn_gap > 4m` 时平滑增长到1.0（更深拱弧）
  - `arc_handle = max(0.5, coef × turn_gap × scale + handle_bias)`
  - `min_lead0 > 1m` 时 `arc_handle = max(arc_handle, 0.3 × min_lead0)`

- **lateral_bias**（横向平移）：
  - 沿U型横向方向对称平移两个内部控制点
  - 保留G1连续，用于扇开共享进入车道的多个U型调头
  - 钳制范围: `tan(15°) × total_handle` (turn_gap < 4m) 或 `tan(30°) × total_handle` (turn_gap ≥ 4m)

### 4.9 多重约束的协同机制

8重约束在以下场景中相互制约，需要综合处理：

#### 4.9.1 避障 vs 同簇非交

**冲突**：曲线为避障向一侧偏移可能挤压同簇兄弟曲线。

**解决**：
- 权重优先级：避障(30) > 簇约束(20) > 围栏(6)
- 避障违反时自适应放大权重至240，簇约束至多96
- `checkAndMarkA2`：若避障导致必须交叉，标记ObstacleCross弱化簇约束至0.4×（非零，保持基本保序）

#### 4.9.2 同簇非交 vs 人行横道跨越

**冲突**：同arm上多个U型调头获得不同人行横道跨越距离会导致拱弧顶点深度不一致，破坏有序排列。

**解决**：
- `crosswalkClearanceAhead` 使用 `far`（远端投影）而非 `forward`（最近点投影）
- 确保同arm上所有U型调头获得一致的跨越距离
- U型调头求解器在网格搜索中同时考虑 `lead0_extra` 与 `lateral_bias`，保持两条拱弧的横向有序

#### 4.9.3 G1连续 vs 同簇非交

**冲突**：满足G1可能迫使曲线进入兄弟曲线区域。

**解决**：
- G1由构造保证（makeCubicG1/makeAlignedUTurnCubic）
- 优化后 enforceG1 强制投影回切向射线
- 簇约束通过控制点偏移而非切向调整，与G1正交

#### 4.9.4 固定形态 vs 避障

**冲突**：固定形态曲线穿障时需重新生成，但约束要求保留形态。

**解决**：
- `fixedGeometryHitsObstacle` 检测固有形态是否穿障
- 仅穿障时允许重新生成，其他场景保留
- 重新生成时仍满足其他所有约束

#### 4.9.5 U型调头多约束求解

**冲突**：U型调头同时受G1、避障、围栏、簇约束、人行横道跨越、最大曲率、拱弧形态多重约束。

**解决**（`solveUTurnMultiConstraint`）：

网格搜索 `scale × lateral_bias × lead0_extra × handle_bias`（约600候选）：
```
cost = 1000 × sib_x            // 同簇交叉数(高权重)
     + 200  × obst_pen         // 障碍物穿透
     + 200  × fence_ovf        // 围栏越界
     + 50   × (1 - g1_min)     // G1偏离
     + 20   × max(maxk-1, 0)   // 最大曲率
     + 5    × |arc/c - 1.55|   // 拱弧形态偏离
     + 0.5  × (|lat_bias|+|handle_bias|+0.3·lead0_extra)  // 紧凑度
     + 0.1  × |arc - ref_len|  // 弧长偏离
```

硬过滤：
- 自相交 → 拒绝
- 超出envelope → 拒绝
- maxk > 物理上限 → 拒绝

#### 4.9.6 生成顺序保证

按"直行→左右转→U型调头"+"由内到外"的顺序生成，确保外侧曲线生成时内侧邻居已就绪，作为兄弟传入携带正确的 `expected_side`。

#### 4.9.7 多轮拓扑修复

最多3轮全局修复：
1. **第1轮**: 纯几何拓扑修复 (`tryPureGeometryTopologyRepair`) — 重试不同参数组合，选交叉最少的
2. **第2轮**: U型调头专用修复 (`generateOne` with `allow_uturn_search=true`)
3. **第3轮**: 标记 `CurveStatus::Degraded` 并记录 `exempt_crosses`

每轮最多修复12条曲线，避免单点故障导致整体重生成。

## 5. 精细路口面生成（IntersectionAreaBuilder）

精细路口面是封闭可内凹多边形，由道路边缘、车道端点、车道边线共同确定。

### 5.1 输入

- 道路边缘线（`input.boundaries`，Type=RoadEdge）
- 进入/退出车道组端侧边界
- 已生成的连通曲线端点
- 已生成的车道边线（用于吸附）

### 5.2 构建流程

```
1. 计算路口中心与半径
   center = 车道端点的几何中心
   radius = max(端点到center的距离)

2. 优先尝试精细多边形构建 (buildFinePolygon)
   - 收集所有车道端点 + 道路边缘在路口侧的端点
   - 按角度排序围成多边形
   - 检测可内凹处(同arm多车道间的空隙) → 内凹处理
   - 若多边形顶点数>=4 → 直接返回

3. 回退到道路边缘裁剪法 (buildFromBoundaryElements)
   for 每条道路边缘 boundary[i]:
     a. 找到路口侧端点(距center更近的端点)
     b. 检查是否在路口范围内 (在粗糙路口面内 或 距center < 1.5×radius)
     c. 吸附到最近的车道边线端点 (snapTolerance=0.5m)
     d. 记录端点 → endpoints[i]

4. 端侧边界回缩/外扩
   for 每个车道组:
     - 进入组: 端侧边界向路口内回缩 groupCutOffset=0.05m
     - 退出组: 端侧边界向路口外扩 groupCutOffset=0.05m
     - 边界折线两端外延 cutLineExtension=0.20m 对齐道路边缘

5. 与道路边缘相交时截取有效段
   - 计算端侧边界外延与道路边缘的交点
   - 截取交点之间的有效段作为路口边线

6. 按顺时针绕向组装多边形 (winding="clockwise")
   - 沿路口外围依次连接有效段
   - 顶点吸附 snapTolerance=0.5m 合并近邻点

7. 返回 IntersectionArea { is_rough = false }
```

### 5.3 关键参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| snapTolerance | 0.5m | 顶点吸附容差 |
| winding | "clockwise" | 多边形绕向 |
| groupCutOffset | 0.05m | 进入组内缩/退出组外扩距离 |
| cutLineExtension | 0.20m | 端侧边界两端外延长度 |

### 5.4 可内凹处理

当同arm存在多车道时，车道间的空隙形成路口面的内凹：
- 收集每条车道端点 + 车道边线在路口侧的端点
- 按角度排序后，相邻车道端点之间插入车道边线端点
- 形成可内凹的多边形外环

### 5.5 异常处理

- 道路边缘线不足 → 回退到凸包
- 端侧边界与道路边缘无交点 → 使用外延端点
- 多边形自相交 → 标记错误，返回粗糙路口面

## 6. 关键算法

### 6.1 Bezier曲线表示

每条连通曲线由若干段三次Bezier段拼接：
```
B(t) = (1-t)³P0 + 3(1-t)²t P1 + 3(1-t)t² P2 + t³ P3
```

**G1连续**：相邻段在连接点切向一致。

### 6.2 LBFGS优化

拟牛顿法，存储最近m步历史：
```
for iter = 1..max_iter:
    g = grad(x)
    if ||g|| < grad_tol: 收敛
    d = -H·g  (双循环计算逆Hessian近似)
    alpha = lineSearch(d)  // Strong Wolfe
    x += alpha·d
```

### 6.3 弹性带平滑

迭代过程：
```
for iter = 1..max_iter:
    for i = 1..N-1:
        k = localCurvature(p[i-1], p[i], p[i+1])
        if k <= kappa_max: continue
        cen = circumcenter(p[i-1], p[i], p[i+1])
        dir = (p[i] - cen).normalized()
        cand = p[i] + dir × move_step
        if sdf(cand) >= min_sdf && polygonContains(fence, cand):
            p[i] = cand
```

## 8. 异常处理

| 异常 | 处理策略 |
|------|---------|
| 拓扑校验失败 | 返回false，输出错误列表，不生成 |
| 通道过窄 | 标记CurveStatus::WarnA2，仍生成最优曲线 |
| 围栏-障碍夹断 | 尝试放松围栏，仍失败则标记Infeasible |
| 拓扑死锁 | 生成fallback直线段，标记Infeasible |
| LBFGS发散 | 保留初始曲线，标记Degraded |
| 同簇交叉无法消除 | 标记Degraded并记录exempt_crosses |
